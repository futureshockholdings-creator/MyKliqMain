## Requirements

### iOS (App Store)
**Required Size:** 1320 × 2868 pixels (iPhone 6.9" display)
- Format: JPEG or PNG
- Max file size: 10 MB
- Quantity: 1-10 screenshots
- Apple auto-scales for all iPhone models

**Optional iPad Size:** 2064 × 2752 pixels (iPad 13" display)

### Android (Google Play)
**Required Size:** 1080 × 1920 pixels (9:16 portrait)
- Format: JPEG or PNG (24-bit, no transparency)
- Max file size: 8 MB
- Quantity: Minimum 2, recommended 4-8 screenshots

**Optional Tablet Size:** 1920 × 1080 pixels (16:9 landscape)

## How to Capture Real Device Screenshots

### For iOS:
1. Run the app on iPhone 16 Pro Max (6.9") or iPhone 15 Pro Max simulator
2. Navigate to each key screen
3. Press `Cmd + S` in simulator to save screenshot
4. Screenshots save to Desktop by default

### For Android:
1. Run the app on Pixel 6/7 or similar device (1080 × 1920 resolution)
2. Navigate to each key screen
3. Use Android Studio's screenshot tool or device screenshot feature
4. Save with proper naming convention

## Best Practices
- Show real UI from the actual app
- Highlight 1-2 key features per screenshot
- Keep text overlays minimal (6-8 words max)
- First 2-3 screenshots are most important
- Use all available slots (up to 10 for iOS, 8 for Android)
- Ensure good lighting and contrast
- Avoid showing sensitive user data
